后端实现方案

选择后端架构实现方式

A

扩展现有FastAPI

在现有backend基础上新增任务管理端点，Agent通过回调推送状态

  • ✓ 改动最小
  • ✓ 与现有系统集成简单
  • ✗ 现有代码耦合较紧
B

任务管理层中间件

新增独立任务管理服务，监听Agent执行日志，实时推送到前端

  • ✓ 架构清晰，扩展性好
  • ✓ Agent代码无需修改
  • ✗ 需要新增服务
C

完全独立服务

新建独立Web服务，完全解耦，只负责任务调度和前端展示

  • ✓ 完全解耦，独立部署
  • ✓ 可以被其他系统复用
  • ✗ 需要API适配器

推荐方案：扩展现有FastAPI + 任务管理层
